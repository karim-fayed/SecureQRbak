# دليل المالك الرئيسي للنظام

## معلومات الحساب

- **البريد الإلكتروني**: `karim-it@outlook.sa`
- **كلمة المرور الافتراضية**: `SecureQR@2024` (يرجى تغييرها بعد أول تسجيل دخول)
- **الدور**: `admin` (مدير النظام)
- **خطة الاشتراك**: `enterprise` (نشطة)

## الصلاحيات الكاملة

المستخدم `karim-it@outlook.sa` لديه صلاحيات كاملة في النظام، بما في ذلك:

### ✅ إدارة المستخدمين
- عرض جميع المستخدمين
- تعديل بيانات المستخدمين
- تغيير أدوار المستخدمين
- تغيير خطط الاشتراك
- حذف المستخدمين

### ✅ إعدادات النظام
- تكوين إعدادات الموقع
- تفعيل/تعطيل وضع الصيانة
- السماح/منع التسجيل
- تحديد الحد الأقصى لرموز QR المجانية
- إعدادات الإشعارات

### ✅ سجلات النظام
- عرض جميع سجلات مسح رموز QR
- إحصائيات الاستخدام
- تتبع الأحداث

### ✅ إحصائيات النظام
- إجمالي المستخدمين
- إجمالي رموز QR
- المشتركون المدفوعون
- عمليات المسح
- توزيع الخطط

### ✅ إدارة الاشتراكات
- تغيير خطط المستخدمين
- تفعيل/إلغاء الاشتراكات
- تعديل تواريخ انتهاء الصلاحية

## كيفية الوصول

1. **تسجيل الدخول**: استخدم البريد الإلكتروني وكلمة المرور
2. **الوصول للوحة التحكم**: بعد تسجيل الدخول، ستظهر رابط "لوحة المالك" في القائمة الجانبية
3. **أو الوصول المباشر**: `/admin`

## صفحات لوحة التحكم

- `/admin` - لوحة التحكم الرئيسية مع الإحصائيات
- `/admin/users` - إدارة المستخدمين
- `/admin/users/[id]` - تعديل مستخدم محدد
- `/admin/system-settings` - إعدادات النظام
- `/admin/logs` - سجلات النظام

## الأمان

### ضمان الصلاحيات
- النظام يضمن تلقائياً أن المالك لديه دائماً صلاحيات `admin`
- حتى لو تم تغيير الدور، سيتم استعادته تلقائياً
- خطط الاشتراك دائماً `enterprise` نشطة

### تغيير كلمة المرور
**مهم جداً**: بعد أول تسجيل دخول، يجب تغيير كلمة المرور الافتراضية:
1. اذهب إلى `/dashboard/settings`
2. استخدم قسم "تغيير كلمة المرور"
3. أدخل كلمة مرور قوية

## السكريبتات المتاحة

### إعداد المالك
```bash
npm run setup-owner
```
هذا السكريبت:
- يتحقق من وجود المالك
- ينشئ الحساب إذا لم يكن موجوداً
- يضمن أن لديه صلاحيات كاملة
- يحدّث كلمة المرور إذا لزم الأمر

### Seed قاعدة البيانات
```bash
npm run seed
```
يؤسس قاعدة البيانات وإنشاء المالك إذا لم يكن موجوداً.

## ملاحظات مهمة

1. **كلمة المرور**: تغيير كلمة المرور الافتراضية فوراً
2. **النسخ الاحتياطي**: احتفظ بنسخة احتياطية من قاعدة البيانات بانتظام
3. **الأمان**: لا تشارك بيانات تسجيل الدخول مع أي شخص
4. **الصلاحيات**: المالك لديه صلاحيات كاملة دائماً، حتى لو تم تغييرها يدوياً

## الدعم

في حالة وجود أي مشاكل، تحقق من:
1. سجلات النظام في `/admin/logs`
2. إعدادات النظام في `/admin/system-settings`
3. حالة قاعدة البيانات في `/admin`
